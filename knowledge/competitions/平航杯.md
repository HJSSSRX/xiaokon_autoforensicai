# 平航杯 题型分类

中国公安部系统电子数据取证竞赛，难度较高，题量大。

---

## 基本信息

- 主办：公安部第三研究所等
- 频次：每年一届
- 形式：线上初赛 + 线下决赛；个人赛 + 团体赛
- 题量：团体赛通常 60-80 题，3-5 小时
- 总分：常见 500 分级别
- 参考题集：2025 年团体赛有公开 writeup；2026 年在 2025 基础上变体

---

## 检材构成（典型）

一场比赛常见 5-8 个检材：

| 检材 | 说明 | 占比 |
|---|---|---|
| 主角 PC 磁盘镜像（E01） | 一个或多个角色的 Windows 机器 | 30-40% |
| Linux 服务器镜像 | 跑 Trojan/Xray/网站/API 后端 | 15-20% |
| 安卓手机备份 zip | 受害者或攻击者的手机 | 15-20% |
| 内存镜像 | 对应某台 PC 的运行时内存 | 10-15% |
| 流量包 pcap/pcapng | 网络或蓝牙 | 10% |
| 恶意样本独立文件 | PE/APK/脚本（可能内嵌在镜像里） | 5-10% |
| 加密容器 | VeraCrypt / BitLocker | 小题附带 |

## 考察方向分布（基于 2025 团体赛 + 2026 预览）

### 1. 基础信息类（~15-20%）
- 手机型号、IMEI、系统版本
- 电脑 Build、计算机名、时区
- 服务器内核版本、用户数
- 微信 wxid、QQ 号
- **工具一把梭**。

### 2. 应用数据类（~15-20%）
- 微信聊天、通话记录、短信
- 浏览历史、搜索关键词
- 地图足迹、照片 EXIF
- 便签、Sticky Notes、日记
- **挂载镜像 + sqlite + 火眼**。

### 3. 恶意软件逆向（~20-30%）—— AI 主场
- .NET 木马分析（反沙箱、持久化、C2、加密）
- JS/VBS/宏 脚本分析（C2 域名、解密密钥、自毁函数）
- APK 自研 app 逆向（密码长度、加密算法、硬编码）
- 可能含 SO 原生库题（硬骨头）

### 4. 服务器与 Web（~10-15%）
- 配置文件（Redis/MySQL/Nginx 密码）
- 后台登录加密复现
- rockyou 字典爆破
- 攻击者 IP 和上传的 webshell
- **脚本行为复现**（Q26-29 那类，读 Python 函数推参数）

### 5. 内存取证（~10%）
- 识别微信进程
- 提取微信数据库密钥（32字节）
- 找恶意进程 PID 和创建时间
- 从内存提取 C2 IP

### 6. AD 域 / 内网（~5-10%）
- BloodHound 图分析
- 权限边识别
- 攻击路径还原
- 域名常用 `XIAORANG.LAB`（春秋云境靶场）

### 7. 社工 / 字典爆破（~5%）
- VeraCrypt 外层密码（从笔记 / 剪贴板推）
- 加密 zip（从朋友圈 / 日记生成字典）
- utools 剪贴板常藏密码

### 8. 流量（~5-10%，不是每届有）
- HTTP 对象还原
- 蓝牙设备伪装分析
- cmd 命令执行还原

---

## 典型考点延续（跨届重复出现）

- 微信数据库解密
- Trojan 服务器配置（remote_addr / forward / server 模式）
- 后台密码加密算法复现（bcrypt + 盐）
- SillyTavern + AI 换脸软件（考角色/模型分析）
- VeraCrypt 容器
- neo4j 数据库密码（默认 `neo4j/neo4j` 或自定义）
- 虚拟货币钱包助记词

## AI 可以吃透的题型（按预期得分率）

| 题型 | 预期得分率 | 原因 |
|---|---|---|
| 基础信息类 | 90% | 工具直接出，读文本 |
| .NET 逆向 | 80-90% | ilspycmd + 读 C# |
| JS 脚本分析 | 80-90% | 去混淆后读代码 |
| 应用数据类 | 70-80% | sqlite 查询，需要人辅助识别 |
| 服务器配置题 | 80% | 读配置文件 |
| 脚本行为复现 | 85% | LLM 读 Python 超强 |
| 内存取证 | 50-70% | 看 symbol table 运气 |
| AD 域分析 | 60% | Cypher 查询不难，理解边含义 |
| APK 逆向 | 50-70% | Java 好读，SO 很难 |
| SO 原生逆向 | 20-40% | 需要动态调试 |
| 蓝牙流量 | 40-60% | 需要蓝牙协议知识 |
| 需要火眼仿真的题 | 依赖人类操作员 | AI 读导出产物 |

## 坑汇总（历届踩过的）

- **火眼仿真蓝屏**：部分镜像启动后蓝屏，因存在 `$OSDATA` 文件，删除后可仿真
- **微信加密**：PC 微信密钥只在进程内存里，一重启就没了。内存镜像必须在微信运行时抓
- **时区不同**：题目要求 UTC，但日志默认本地时间，差 8 小时
- **多个 uin/wxid**：一个手机可能登录过多个微信，注意取题目问的那个
- **ALEAPP 不支持国产 app**：微信/QQ/支付宝要手动挖
- **.NET 混淆**：ConfuserEx/Eazfuscator 常见，需要 `de4dot` 先脱
- **APK 加固**：乐固/360/爱加密，静态字符串看不到真内容
- **neo4j 版本差异**：导入 BloodHound 数据时版本不兼容会报奇怪错
- **VeraCrypt 慢**：`hashcat -m 13721` 爆破极慢，字典必须小而精
- **鸣谢型 writeup**：CSDN 上 writeup 常有错答案，交叉验证再采信

## 3 小时团体赛推荐战术

1. **0-30min**：并行跑所有预处理（ALEAPP / EZ / fls / vol pslist）
2. **30-90min**：速刷基础信息 + 工具直出题（目标 150-180 分）
3. **90-150min**：主攻逆向题（.NET + JS + APK Java 层）（目标 +120-150 分）
4. **150-180min**：内存 + 域 + 流量 + 抢救题（目标 +50-80 分）

**三机分工建议**见 `/PLAYBOOK.md` 和项目 README。
