## 关键技术沉淀

### 1. MIUI bak 解开三步法

```bash
# 头部 = "MIUI BACKUP\n2\n<pkg> <appname>\n-1\n0\nANDROID BACKUP\n3\n0\nnone\n"
# 之后是 tar 流（无压缩）
OFFSET=$(grep -aboP '^apps/' file.bak | head -1 | cut -d: -f1)
dd if=file.bak of=tmp.tar bs=4K iflag=skip_bytes skip=$OFFSET status=none
tar -C out_dir -xf tmp.tar
```

### 2. 中文文件名 zip 解压

```bash
unzip -O CP936 -o backup.zip   # MIUI 备份默认 GBK 编码
```

### 3. AES whitebox 反编译流程

- 看 jadx 反编译的 Java 类：找 `MAGIC_NUMBERS` / `CIPHER_PART*` 等常量
- 推断 key 派生：`MAGIC_NUMBERS XOR <const>`
- 注意 fallback 分支（如 `hexStringToByteArray` catch 返回固定字节）—— 这往往就是真正密文
- pycryptodome AES.MODE_ECB + PKCS#7 unpad 即可

## 踩坑

1. **PowerShell 中调用 `bash -lc "..."`** 双引号嵌套频繁失败。最稳：把 bash 脚本写到 `cases/2025平航杯/artifacts/m_*.sh`，用 UTF-8 (no BOM) 写入并去 `\r\n`，然后 `wsl bash /path/to.sh`。
2. **head 命令冲突**：PowerShell 没有 `head`，所以管道 `wsl ... | head -10` 会失败。把 `head` 也放到 bash 脚本里。
3. **MIUI bak 头部有 2 行 包名 / 应用名**：用 `head -c 200 | strings | grep -E '^com\.'` 抓不到中文带空格的应用名行。改用 `^apps/` 锚点找 tar 起点更可靠。
4. **梆梆加固静态死路**：lib SecShell + 加密 classes0.jar，纯静态搜不到 IP/port/字符串。要解 M08-M10 必须 ARM 模拟器 + frida-dexdump（这次跳过）。
5. **PuzzleGame APK 多个 dex (classes.dex - classes6.dex)**：业务代码分散，jadx 多 dex 解包 OK；剧情图 `pic1.jpg` 只一张；干扰类 `Decryptor.java` / `reDecryptor.java` 显式带 FAKE_ 前缀。
