## 关键技术沉淀

### 1. WSL2 挂载 vmdk + LVM + XFS 三件套

```bash
# 1. 装工具
sudo apt-get install -y qemu-utils libguestfs-tools mariadb-server

# 2. 挂 vmdk (libguestfs 在 WSL2 因 KVM 不可用会失败 → 走 nbd)
sudo modprobe nbd max_part=16
sudo qemu-nbd -r -c /dev/nbd0 /path/to/disk.vmdk

# 3. 激活 LVM
sudo vgchange -ay cl
sudo dmsetup mknodes  # WSL2 udev 不工作，需手动建节点

# 4. XFS 必须 norecovery（read-only 不能日志重放）
sudo mount -o ro,norecovery /dev/mapper/cl-root /mnt/server
sudo mount -o ro /dev/nbd0p1 /mnt/server_boot
```

### 2. mysql binlog 绕过启动 mysqld 取数据

适用场景：表 .ibd 已删但 binlog 还在。

```bash
# MariaDB 10.6 的 mysqlbinlog 兼容 MySQL 5.6 binlog
sudo mysqlbinlog --base64-output=DECODE-ROWS -v file.bin > out.sql
# 注意：--skip-gtids 参数 MariaDB 不支持，去掉

# binlog 是 statement-based，可直接 grep INSERT
grep -F 'INSERT INTO `tp_order`' out.sql | wc -l
```

Python 解析 INSERT VALUES：
```python
parts = re.findall(r"NULL|'(?:[^'\\]|\\.)*'", values_clause)
# 第 N 列取 parts[N-1].strip("'")
```

### 3. PowerShell + WSL bash 混用规约

```powershell
wsl -d Ubuntu -- bash "/mnt/f/cloud/DD/$($path.Replace('\','/'))"
```

## 踩坑

1. **libguestfs guestmount 在 WSL2 失败**：因为依赖 KVM，supermin 起不来。改 qemu-nbd。
2. **WSL2 LVM /dev/mapper 节点不自动创建**：`vgchange -ay` 后还需 `sudo dmsetup mknodes`。
3. **XFS read-only 不能直接挂**：要加 `norecovery` 否则报 "log mount failed"。
4. **mysqlbinlog 旧参数**：`--skip-gtids` 在 MariaDB 10.6 不存在；别带。
5. **数据库被删但 ibdata1 还在**：表元数据 .frm/.ibd 都没了，但 binlog 是真理之源。
6. **bash 反引号在 grep 中**：`grep 'INSERT INTO \`tp\`'` 失败 → 改 `grep -F 'INSERT INTO `tp`'` 或不用反引号。
