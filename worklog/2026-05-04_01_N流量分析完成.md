## 📚 关键技术要点

### BLE 分析
- `capinfos` 读 pcapng IDB 元数据（`Interface #0 Name` = 抓包设备名）
- `tshark -Y 'btcommon.eir_ad.entry.device_name'` 提取 Complete Local Name
- `tshark -Y 'btcommon.eir_ad.entry.company_id'` 提取 Manufacturer Specific Data
- Flipper Zero 伪装特征：同一硬件，MAC 第 3 字节差 1（`80:e1:26:33:32:31` ↔ `80:e1:26:35:32:31`）
- BLE 3 信道重复抓包 → 去重前先按时间排序再 sort

### USB HID 解码
- device 2.4 ITE 048d:c102（联想内置键盘）endpoint 1 = 键盘 interrupt IN
- HID 报告 37 字节（非标 8 字节）：Report ID (1) + Modifier (1) + Reserved (1) + Keys[6] + Padding[28]
- **CapsLock 状态机必须实现**：`<CAPS>` 每按一次切换状态，不能当字符输出
- Shift + CapsLock XOR：`upper = caps_on XOR shift`
- 只处理 Key Press（新 keys）而非 Release

### 环境踩坑
- PowerShell 写 bash 脚本 → CRLF 污染，必须用 `[System.IO.File]::WriteAllText(..., $s.Replace("`r`n","`n"), utf8_nobom)`
- 中文路径 `F:\TEXT(important)\Jian cai\2025平航杯\` → WSL `'/mnt/f/TEXT(important)/Jian cai/2025平航杯'`（单引号避免空格和括号）
- `bash -lc '...'` vs `bash -lc "..."` 引号嵌套复杂时改用 heredoc 或 PS 写脚本 → `wsl bash /mnt/f/...`
