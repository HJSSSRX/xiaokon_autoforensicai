# WP_FORMAT — 每 5 题答案的规范化输出

> 每做满 5 题，Cascade **必须**生成一个 `wp_batches/QNN-QMM.md` 文件。
>
> **用途**：赛后把所有批次 `cat` 拼起来 → 即可作为完整 Writeup 提交。
>
> **强制字段**：题号、题目原文、答案、解题方法、证据路径、不作弊证明、置信度。

---

## 📄 批次文件命名

```
cases/<比赛>/wp_batches/[A|B|C]-Q<起>-Q<止>.md
```

- `[A|B|C]-` 前缀用于三机协作（只本机操作时可省）
- 题号用零填充两位，如 `Q01-Q05.md`、`Q26-Q30.md`
- **严禁**一个文件包含超过 5 题（赛后拼 WP 时要按节分）

---

## 🧩 单题模板（每道题都用这个）

复制 `_template.md` 即可：

```markdown
## Q<NN> — <题目简称>

**题目原文**：
> <完整复制 questions.xlsx 的原文，一字不改>

**答案**：`<最终提交值>`
> 格式说明（若有）：<例：MAC 地址必须小写冒号分隔>

**置信度**：**高 / 中 / 低**
> 理由：<为什么是这个置信度。低置信必须说有哪些没排除的可能>

---

### 解题方法

1. **识别**：<这题属于哪类检材？需要什么 artifact？>
2. **提取**：
   ```bash
   # 实际跑过的命令（复制粘贴级，含路径）
   <cmd>
   ```
3. **分析**：<从输出中找到什么？如何推出答案？>
4. **验证**：<用什么二次确认？>

### 关键证据

- **证据路径**：`<绝对路径>` 的第 N 行 / offset X
- **证据片段**（必要时贴 10 行以内原文）：
  ```
  <raw>
  ```

### 不作弊声明

- 数据来源：**仅** `<检材路径>`（只读挂载）
- 未访问：外部搜索引擎、写题解博客、AI 平台（推理 ≠ 查答案）
- 工具：<列出用的开源工具和版本号>
- 推导链可复现：本批次所用脚本在 `artifacts/<题号>_*.sh` 留存
```

## 🔧 自动生成脚本（推荐）

用户在 Windsurf 里说 "生成 Q06-Q10 批次"，Cascade 应：

1. 从 `answers.md` 拉对应题号的已记录答案
2. 从 `questions.md` 拉原题文本
3. 填入本模板
4. 写到 `wp_batches/X-Q06-Q10.md`
5. 更新本批次 TASKS.md 进度条
6. 若缺字段（如证据路径），**停下**问用户，不许凭空写

---

## ⚠ 死规矩

| 规矩 | 违反后果 |
|------|---------|
| 每题必有「不作弊声明」 | 赛后评委质疑无法自证 |
| 每题必有「证据路径」 | 后期对不上号重做 |
| 未解题**必须**写原因（不许省略） | 下次 AI 看不懂上次卡在哪 |
| 不许在批次文件里写"见 xxx.md"式跳转 | 失去自包含性 |
| 答案格式必须**完全匹配**题目要求 | 正确答案因格式扣分 |
