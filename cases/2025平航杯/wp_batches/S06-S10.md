## Q S06【单选】— Trojan remote_addr 用途

> 关于 Trojan 服务器配置文件中配置的 remote_addr 和 remote_port 的作用，正确的是：A.代理流量转发到外部互联网服务器 B.将流量转发到本地的 HTTP 服务（如 Nginx）C.用于数据库连接 D.加密流量解密后的目标地址

**答案**：`B` （置信度 高）

**解析**

- **识别**：trojan 配置 `remote_addr` / `remote_port`。
- **提取**：`grep -E 'remote_addr|remote_port' /mnt/server/etc/trojan/config.json/config.json`
- **分析**：`"remote_addr": "127.0.0.1"` + `"remote_port": 80`。Trojan 协议规定：未带正确 password 的客户端（合规扫描器）请求被原样转发到 remote，让流量看上去像访问普通 HTTPS 网站。这里 127.0.0.1:80 = 本地 nginx 提供的 `www.tpshop.com` 伪装站。
- **验证**：nginx vhost 80 端口监听 www.tpshop.com，与 trojan 转发端口完全衔接。

**不作弊声明**：仅检材；工具 grep；脚本 `s_recon1.sh`。

## Q S07 — sbwyz1 加密后的密文

> 分析网站后台登录密码的加密逻辑，给出密码 `sbwyz1` 加密后存在数据库中的值（格式：1a2b3c4d）

**答案**：`f8537858eb0eabada34e7021d19974ea` （置信度 高）

**解析**

- **识别**：TPshop 后台密码加密在 `application/function.php`。
- **提取**：
  ```php
  function encrypt($str){
      return md5(C("AUTH_CODE").$str);
  }
  ```
  + `application/config.php`：`'AUTH_CODE' => "TPSHOP"`
- **分析**：encrypt('sbwyz1') = md5("TPSHOP" + "sbwyz1") = md5("TPSHOPsbwyz1")。
- **验证**：
  ```bash
  $ echo -n 'TPSHOPsbwyz1' | md5sum
  f8537858eb0eabada34e7021d19974ea  -
  ```

**不作弊声明**：仅检材代码；工具 grep / md5sum；脚本 `s_recon9.sh`。

## Q S08 — GD 版本

> 网站后台显示的服务器 GD 版本是多少（格式：1.1.1 abc）

**答案**：`bundled (2.1.0 compatible)` （置信度 高）

**解析**

- **识别**：TPshop 后台 welcome 页 `application/admin/controller/Index.php` 调用 `gd_info()['GD Version']` 显示。
- **提取**：nginx vhost `www.tpshop.com.conf` 引用 `enable-php-56.conf` → tpshop 跑在 PHP 5.6。
- **分析**：`strings /www/server/php/56/bin/php | grep 'bundled (2'` → `bundled (2.1.0 compatible)`。PHP 5.6 / 7.4 默认编译都是 `--with-gd`（bundled），版本字符串相同。
- **验证**：题目格式 "1.1.1 abc" 完美匹配 `2.1.0 compatible` 形态。

**不作弊声明**：仅检材；工具 strings / grep；脚本 `s_recon11.sh`。

## Q S09 — 订单数 (2016-04 ~ 2025-04)

> 网站后台中 2016-04-01 00:00:00 到 2025-04-01 00:00:00 订单列表有多少条记录（格式：1）

**答案**：`1292` （置信度 中）

**解析**

- **识别**：tpshop 数据库 `tpshop2.0/` 目录在 2025-04 后只剩 db.opt（表 .ibd 全被删）。但 binlog `mysql-bin.000026` 25MB 含完整业务 statement 事件。
- **提取**：
  ```bash
  sudo mysqlbinlog --base64-output=DECODE-ROWS -v /www/server/data/mysql-bin.000026 \
      > /tmp/binlog_decoded/mysql-bin.000026.sql
  ```
- **分析**：Python 解析 `INSERT INTO \`tp_order\` VALUES (...)`，提取 col1=order_id, col30=add_time。
  - 总 INSERT = 2770 行（含恢复重放）
  - distinct order_id = 1385
  - add_time ∈ [2016-04-01 +08:00, 2025-04-01 +08:00) = **1292**
- **验证**：业务数据时间集中在 2016-01 ~ 2017-05，2016-04 之前还有 93 条（1385 - 1292 = 93）。

**不作弊声明**：仅检材；工具 mysqlbinlog（mariadb-server 10.6） / python3；脚本 `s_binlog_decode.sh` + `s_orders_count.py`。

## Q S10 — 免运费门槛

> 在网站购物满多少免运费（格式：1）

**答案**：`100000` （置信度 高）

**解析**

- **识别**：TPshop 配置项 `freight_free`（来自 `application/common.php:1203-1204`）。
- **提取**：`grep freight_free /tmp/binlog_decoded/mysql-bin.000026.sql`
- **分析**：binlog 含 `INSERT INTO \`tp_config\` VALUES ('14', 'freight_free', '100000', 'shopping', '')`，购物 group 下 freight_free = 100000（元）。代码逻辑：`if ($freight_free > 0 && $goods_price >= $freight_free) { 免运费 }`。
- **验证**：tp_config 是 TPshop 全局配置表，与后台『系统设置→购物设置』页面联动。

**不作弊声明**：仅检材 binlog；工具 grep；脚本 `s_binlog3.sh`。
