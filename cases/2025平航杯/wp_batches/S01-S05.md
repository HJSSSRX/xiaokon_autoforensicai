## Q S01 — 服务器最早开机时间

> 该电脑最早的开机时间是什么（格式：2025/1/1 01:01:01）

**答案**：`2022/2/23 12:23:49` （置信度 高）

**解析**

- **识别**：`/var/log/wtmp` 二进制日志记录系统启动 / 用户登录历史。
- **提取**：`sudo last -f /mnt/server/var/log/wtmp -F | tail -1`
- **分析**：输出 `reboot system boot 3.10.0-514.el7.x Wed Feb 23 12:23:49 2022`，文件页脚 `wtmp begins Wed Feb 23 12:23:49 2022`，即 wtmp 中最早的一条 reboot 记录。
- **验证**：`/etc/redhat-release` Birth time 与 `/root` 目录 Birth time 都在 2022-02-23 11:35-12:23 时段，与 wtmp 一致。

**不作弊声明**：仅访问检材；工具 last / qemu-nbd / mount；脚本 `cases/2025平航杯/artifacts/s_recon2.sh`。

## Q S02 — 服务器内核版本

> 服务器操作系统内核版本（格式：1.1.1-123）

**答案**：`3.10.0-1160.119.1.el7.x86_64` （置信度 中）

**解析**

- **识别**：`/lib/modules/` 下含每个已安装内核的模块目录。
- **提取**：`ls /mnt/server/lib/modules/`
- **分析**：列出两个版本：
  - `3.10.0-514.el7.x86_64`（装机原始版）

  CentOS 7.9.2009 GA 内核线 = 3.10.0-1160.x。grub.cfg 默认启动后者。
- **验证**：`/etc/redhat-release` = `CentOS Linux release 7.9.2009 (Core)`。

**不作弊声明**：仅检材；工具 ls / cat；脚本 `s_recon1.sh`。备选格式 `3.10.0-1160` 更贴 `1.1.1-123` 模板。

## Q S03 — 非系统用户数

> 除系统用户外，总共有多少个用户（格式：1）

**答案**：`2` （置信度 中）

**解析**

- **识别**：CentOS 7 起 `UID_MIN=1000`，UID >= 1000 即非系统默认用户。
- **提取**：`awk -F: '$3>=1000 && $3<65534 {print $1,$3}' /mnt/server/etc/passwd`
- **分析**：输出 `www 1000` 与 `mysql 1001`，二者均 nologin 服务账户但属宝塔面板创建的非默认账户。root 是系统超管不计。
- **验证**：`/home/` 仅有 `www` 子目录。

**不作弊声明**：仅检材；工具 awk；脚本 `s_recon1.sh`。

## Q S04 — Trojan 混淆域名

> 分析起早王的服务器检材，Trojan 服务器混淆流量所使用的域名是什么（格式：xxx.xxx）

**答案**：`www.wyzaizaoqi.com` （置信度 高）

**解析**

- **识别**：Trojan 配置 `/etc/trojan/config.json/config.json` 的 `ssl.cert` 字段指向证书路径。
- **提取**：`cat /mnt/server/etc/trojan/config.json/config.json`
- **分析**：
  ```json
  "ssl": {
    "cert": "/etc/letsencrypt/live/www.wyzaizaoqi.com/fullchain.pem",
    "key":  "/etc/letsencrypt/live/www.wyzaizaoqi.com/privkey.pem"
  }
  ```
  Let's Encrypt 标准目录结构 `live/<domain>/`，所以混淆域名 = `www.wyzaizaoqi.com`（『王要早起』拼音 = wyz aizao qi）。
- **验证**：与剧情『起早王』化名呼应；run_type=server 时 trojan 用此证书伪装成合法 https 网站。

**不作弊声明**：仅检材；工具 cat / python3；脚本 `s_recon1.sh`。

## Q S05【单选】— Trojan 运行模式

> 分析起早王的服务器检材，Trojan 服务运行的模式为：A.forward B.nat C.server D.client

**答案**：`C` （置信度 高）

**解析**

- **识别**：Trojan 的 `run_type` 字段指明运行模式。
- **提取**：`grep run_type /mnt/server/etc/trojan/config.json/config.json`
- **分析**：第一行 `"run_type": "server"` → 选 C。server 模式监听 443 接受 trojan 客户端流量；非合规请求按 remote_addr 转发伪装。
- **验证**：与 S06 配合 — remote_addr=127.0.0.1:80 是『伪装目标』，证明是 server 模式。

**不作弊声明**：仅检材；工具 grep；脚本 `s_recon1.sh`。
