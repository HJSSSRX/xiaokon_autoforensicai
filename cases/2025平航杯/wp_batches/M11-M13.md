## Q M11 — 持久化 API

> 分析倩倩的手机检材，木马 APK 通过调用什么 api 实现自身持久化（格式：JobStore）

**答案**：`JobScheduler` （置信度 高）

**解析**

- **识别**：`aapt dump xmltree base.apk AndroidManifest.xml` 查看 service / permission。
- **提取**：发现 `BIND_JOB_SERVICE` 权限及 `*JobService` 服务声明。
- **分析**：Android 5.0+ 推荐的持久化方案 `android.app.job.JobScheduler`。`BIND_JOB_SERVICE` 是 JobScheduler 框架的特征权限。
- **验证**：与梆梆加固 + native lib 唤起逻辑配合，重启后通过 JobScheduler 恢复 C2 心跳。

**不作弊声明**：本地检材；工具 aapt。

## Q M12 — 倩倩身份证户籍

> 分析倩倩的手机检材，根据倩倩的身份证号请问倩倩来自哪里（格式：北京市西城区）

**答案**：`上海市徐汇区` （置信度 高）

**解析**

- **识别**：解 `com.baidu.input_mi` (百度输入法) 备份；MIUI 备份的 `sp/` 通常保留剪贴板/记忆数据。
- **提取**：`sp/com.baidu.input_mi_preferences.xml` 中 `key_front_clip_content` 字段值 = `310104200108110624`（百度输入法记录的剪贴板上一条内容）。
- **分析**：身份证号 18 位拆解：
  - 前 6 位 `310104` = **上海市徐汇区** 行政区划码
  - 中 8 位 `20010811` = 出生日期 2001-08-11
  - 后 4 位 `0624` = 顺序码 + 校验码
  - 校验位计算：权重 [7,9,10,5,8,4,2,1,6,3,7,9,10,5,8,4,2]，加权和=173，173 mod 11 = 8 → 校验表[8]='4' ✓
- **验证**：剪贴板被输入法明文记录是常见隐私泄露点。

**不作弊声明**：本地检材；工具 grep；脚本 `cases/2025平航杯/artifacts/m_search_id.sh`。

## Q M13 — 手机 IMEI

> 此手机检材的 IMEI 号是多少（格式：1234567890）

**答案**：`865372026366143` （置信度 高）

**解析**

- **识别**：MIUI 系统应用通过 `mistat` (小米统计 SDK) 上报设备指纹，含 IMEI。
- **提取**：`com.android.email/sp/mistat.xml` 中匹配 `imei">865372026366143`。
- **分析**：15 位数字，符合 IMEI 格式。
- **验证**：Luhn 校验：奇数位求和 + 偶数位×2 后逐位求和 = 70，mod 10 = 0 ✓ IMEI 校验通过。

**不作弊声明**：本地检材；工具 grep。
