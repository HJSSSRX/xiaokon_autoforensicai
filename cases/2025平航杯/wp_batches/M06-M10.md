## Q M06 — 木马应用名

> 分析倩倩的手机检材，检材内的木马 app 的应用名称是什么（格式：Baidu）

**答案**：`Google Service Framework` （置信度 高）

**解析**：MIUI 备份文件名 `Google Service Framework(com.example.reverseshell2).bak` 直接揭示。`aapt dump badging` 可双重确认 `application-label`。木马伪装成 Google 系统服务以躲避用户察觉（真实 GSF 包名是 com.google.android.gsf）。

**不作弊声明**：本地检材；工具 unzip / aapt。

## Q M07 — 木马加固类型

> 分析倩倩的手机检材，检材内的木马 app 的使用什么加固（格式：腾讯乐固）

**答案**：`梆梆加固` （置信度 高）

**解析**

- **识别**：解 base.apk 看 lib/ 和主 dex。
- **提取**：`lib/armeabi-v7a/libSecShell.so`、`libSecShell-x86.so`；主 dex 含 `com.SecShell.SecShell.*` stub 类。
- **分析**：`libSecShell` + `com.SecShell` 命名空间是**梆梆加固 (Bangcle SecShell)** 的特征。腾讯乐固用 `com.tencent.StubShell`，爱加密用 `com.shell.NativeApplication`，命名差异显著。
- **验证**：assets/classes0.jar 加密（无 ZIP 头），符合梆梆将原 dex 加密保存的行为。

**不作弊声明**：本地检材；工具 strings / unzip。

## Q M08 — 木马 C2 IP+端口

> 分析倩倩的手机检材，检材内的木马软件所关联到的 ip 和端口是什么（格式：127.0.0.1:1111）

**答案**：**未解**（推断 IP=192.168.180.107，端口需脱壳后确认） （置信度 低）

**解析**

- **识别**：业务代码加密在 `assets/classes0.jar` 和 `r/.cache/classes.jar` 中（前 16 字节随机）。
- **提取**：主 dex 仅含 SecShell 壳代码；strings 主 dex / native lib / 加密 jar 均无 IP 字面量。
- **分析**：M04 已知木马下载源 IP = 192.168.180.107（同局域网），C2 服务器极可能是同一主机，但端口未知。
- **验证（缺）**：需 frida-dexdump 或 BlackDex 在 ARM Android 模拟器实时脱壳。

**不作弊声明**：未联网查询 writeup；工具 strings / aapt / jadx。**未解原因**：梆梆加固静态无法还原 Socket 字面量。

## Q M09 — 偷拍照片数

> 该木马 app 控制手机摄像头拍了几张照片（格式：1）

**答案**：**未解**（手机端备份内无 reverseshell2 运行时数据，需服务器/电脑板块回填） （置信度 低）

**解析**：备份目录 `apps/com.example.reverseshell2/` 仅含 `a/base.apk` 和 `r/.cache/{classes.dex,jar,dve}`（皆加密），无 sp/、f/、db/。意味着木马要么从未运行要么数据已被清理，照片只能在 C2 接收端找到。

**不作弊声明**：本地检材；未联网。**未解原因**：跨板块依赖（S 服务器 / C 计算机），需后续回填。

## Q M10 — 木马所用摄像头

> 木马 APP 被使用的摄像头为（格式：Camera）

**答案**：**未解**（需脱壳确认 Camera/Camera2 API 或前后置） （置信度 低）

**解析**：AndroidManifest 申请 `android.permission.CAMERA` 但具体 API 调用埋藏在加密的 classes0.jar 中。静态无法判断。

**不作弊声明**：本地检材；工具 jadx。**未解原因**：同 M08 — 梆梆加固。
