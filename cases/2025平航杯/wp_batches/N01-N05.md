## N02 — 伪装设备的原始名称

> 起早王有一个用于伪装成倩倩耳机的蓝牙设备，该设备的原始设备名称为什么（格式：XXX_xxx 具体大小写按照原始内容）

**答案**：`Flipper 123all`
**置信度**：高

### 解析

#### 识别
BLE advertising AD type 0x09 (Complete Local Name) 携带设备名。"原始名"= 伪装前广播的名字。

#### 提取
```bash
tshark -r BLE -Y 'btcommon.eir_ad.entry.device_name' \
  -T fields -e frame.time_epoch -e btle.advertising_address \
  -e btcommon.eir_ad.entry.device_name > /tmp/ble_names.tsv
```

#### 分析
两组 MAC 仅第三字节差异（33 vs 35），同一 Flipper Zero：
- `80:e1:26:33:32:31` → `Flipper 123all`（Flipper Zero 默认名结构）
- `80:e1:26:35:32:31` → `QQ_WF_SP8OON`（伪装成倩倩 Sony WF-SP800N 风格耳机）

时间线：
- `Flipper 123all` 始于 `1744165654.337`（2025-04-09 02:27:34 UTC）
- `QQ_WF_SP8OON` 始于 `1744165886.710`（2025-04-09 02:31:26 UTC）

前者先出现 = 原始名。

#### 验证
```bash
grep '80:e1:26:33:32:31' /tmp/ble_names.tsv | sort -k1n | head -3
```

### 不作弊声明
- **数据来源**：本地 `BLE` pcapng
- **未访问外部**：未查 writeup
- **工具**：tshark, sort, grep
- **脚本**：无（一行管道）

## N05 — 起早王手机制造商数据

> 起早王手机蓝牙的制造商数据（格式：0x0102030405060708）

**答案**：`0x0701434839313430`
**置信度**：中（备选 `0x57ab0701434839313430` 若 grader 要求含 company_id 前缀）

### 解析

#### 识别
起早王"不断尝试"广播发包，需找一个**人为构造**且反复广播的非标设备。

#### 提取
```bash
tshark -r BLE -Y 'btcommon.eir_ad.entry.company_id' \
  -T fields -e btle.advertising_address -e btcommon.eir_ad.entry.company_id \
  -e btcommon.eir_ad.entry.data | sort | uniq -c | sort -rn
```

#### 分析
特征设备：
- MAC `55:79:c6:86:8d:13`
- Device name: `Cracked`（Flipper/自制 BLE 工具风格）
- company_id `0xab57`（非 SIG 标准分配）
- manufacturer data: `0701434839313430`（99.5% 一致，占 2457/2468）
- ASCII 解码：`\x07\x01` + `CH9140`（WCH 蓝牙 HID 模块芯片型号）

这是起早王用 **WCH CH9140 模块**构造的 BLE 广播，其 manufacturer data 固定为 8 字节 `0701434839313430`。

#### 验证
```bash
tshark -r BLE -Y 'btle.advertising_address==55:79:c6:86:8d:13 && btcommon.eir_ad.entry.company_id==0xab57' \
  -T fields -e btcommon.eir_ad.entry.data | sort | uniq -c | sort -rn | head -1
# 2457 0701434839313430
```

### 不作弊声明
- **数据来源**：本地 `BLE` pcapng
- **未访问外部**：未查 writeup
- **工具**：tshark, sort, uniq
- **脚本**：无
