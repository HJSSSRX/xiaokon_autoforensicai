# MODE_COOP — 人机协作模式

> 适用：一人一机，题目**复杂或陌生**，要**高质全局**，值得 10 分钟 triage 换后续更稳的推进。
> 核心：AI 主动 triage → 与用户结构化问答补齐已知信息 → 按"独立→依赖→硬骨头"三桶顺序攻题。

---

## 开场 4 阶段

### 阶段 1：环境 + 检材扫描（3 分钟，AI 自动）

```
1. 跑 scripts\env_healthcheck.ps1   → 确认工具齐
2. ls -laR cases/<比赛>/evidence/    → 摸清检材清单
3. 读 questions.md                   → 一次读完全部题目
```

阶段 1 输出：一张**检材-题目映射表**（AI 内存里）。

### 阶段 2：向用户 triage 提问（3 分钟，**一次问完**）

AI **一次性**发这段（不要一句一句问）：

```markdown
## Triage 报告

**检材清单**：
- A:\1-手机\早起王手机.tar
- A:\2-计算机\倩倩PC.E01
- ...

**题目分类**（共 N 题）：
- 安卓板块：Q01-Q17 (17 题)
- Windows 磁盘：Q18-Q37 (20 题)
- 内存：Q67-Q71 (5 题)
- 恶意样本：Q38-Q48 (11 题)
- 跨检材关联：Q49-Q66 (18 题)

## 需要你提供的信息（一次填完最好）

请对下列问题**直接回答**，不必展开：

A. **优先级方向**
   - 必做：______ （题号段或板块）
   - 不做：______ （明确放弃的）
   - 随缘：______（如果时间够就做）

B. **已知答案或部分线索**（有多少写多少）
   - Q<NN>：______（你已经知道的答案或线索）
   - Q<MM>：______

C. **密码/密钥清单**（常见需要的）
   - VeraCrypt 密码：______
   - 压缩包密码：______
   - 微信 IMEI / UIN：______
   - 用户系统密码：______
   - 其他：______

D. **时限**：你期望在多少小时内完成？______

E. **火眼可用性**：火眼证据分析 / 移动取证 / 仿真，分别可用吗？______
```

### 阶段 3：AI 分桶（2 分钟，AI 自动）

AI 拿到用户答案后，把**所有题**分成三桶：

| 桶 | 标准 | 先做 |
|----|------|------|
| ✅ **独立可解** | 检材齐、不依赖其他题答案、不需要爆破/逆向 | 立刻做 |
| 🟡 **有依赖** | 需要等某题答案 / 需要用户给密码 / 需要火眼人工 | 做完独立桶后做 |
| 🔴 **硬骨头** | 大型逆向、密码爆破、加密备份 | 如果有时间且分值高 |

把三桶写到 `cases/<比赛>/TRIAGE.md`，然后向用户**确认**：
```
已分桶：独立 25 题 / 依赖 30 题 / 硬骨头 16 题
建议顺序：独立先→回来做依赖→硬骨头视剩余时间
若 OK 我即刻开始独立桶的 Q01-Q05
```

### 阶段 4：执行（按桶推进）

- 按桶内题号升序做
- 每做完 5 题产出 `wp_batches/Q<NN>-Q<MM>.md`
- 每做完一个**桶**向用户汇报一次（桶内不汇报）
- 独立桶全做完 → 回头扫依赖桶：此时很多"依赖"已被独立桶答案解锁

## 交互风格

### AI 何时问用户（COOP 比 SOLO 多问，但有规矩）

| 情景 | 问法 | 频率上限 |
|------|------|---------|
| 发现新的跨题依赖（某题答案需要另一题的产物） | "Q38 需要 Q10 的 IMEI，我现在 Q38，建议先跳去做 Q10 吗？" | 立刻 |
| 题目真有歧义 | 列 2-3 解读让选 | 每题最多 1 次 |
| 某题答案怀疑用户已有线索 | "Q52 看起来需要日记.docx 密码，你之前提过生日相关？" | 整场赛不超过 5 次 |
| 做到一半发现方向错 | "目前解法需要 8 小时，建议换策略到 YY，耗时 30 分钟但置信低" | 立刻 |

### 何时不打扰用户（COOP 依然要克制）

- 命令输出只要能看懂就自己推，不要"我跑了这个命令，输出如下，你看是什么意思？"
- 工具安装、路径修正这类问题自己解决
- 脚本报错先自己 debug 10 分钟再求助

## COOP 独有：答案汇入 `answers.md` 的前置写入

用户在阶段 2 给了"已知部分答案"后，AI 立刻把这些答案写进 `answers.md`，标记 `[用户提供，待验证]`。做题时若命中再补证据升级到 `[已验证]`。

模板：
```markdown
## Q07 微信最后聊天对象
**答案**：张三  `[用户提供，待验证]`
**证据**：待补
**置信度**：用户自称高
```

## 桶状态看板（AI 自己维护，放 `cases/<比赛>/TRIAGE.md`）

```markdown
# TRIAGE 状态 — 2026-04-25 14:00

## ✅ 独立桶 (25 题)
- [x] Q01 ✅ 已解（答案写入 answers.md）
- [x] Q02 ✅
- [ ] Q03 🔄 做中
- [ ] Q04
- ...

## 🟡 依赖桶 (30 题)
- [ ] Q38 → 等 Q10 的 IMEI
- [ ] Q52 → 等用户提供日记密码
- ...

## 🔴 硬骨头 (16 题)
- [ ] Q14 fpt NAPI 反汇编
- [ ] Q67-Q71 内存 dump
```

每桶推进时 **复制粘贴** 这张表回用户聊天里，让他看到全局。

---

## 弃权指标（COOP 比 SOLO 宽松）

- 卡 40 分钟无进展
- 需要离线爆破 > 3 小时
- 评估单题投入 > 15% 总时间

弃权前在 TRIAGE.md 标记 `⛔ [题号] 原因：...`，最后归入 `answers.md` 的未解区。

---

## COOP 模式 AI 性格调校

- **先摸全局再动手**：triage 没做完不写单题答案
- **结构化问答**：一次问完比分次问效率高 3 倍
- **桶内安静**：进入"独立桶执行"后每 5 题才汇报一次
- **依赖显式**：每个依赖写成 "X → 等 Y"，不要暗中卡住
