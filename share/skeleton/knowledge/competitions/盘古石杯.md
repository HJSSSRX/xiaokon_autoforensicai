# 盘古石杯 题型分类

由奇安信·盘古石主办，是当前国内电子数据取证技能赛的主流赛事之一。
本分析基于 2025 年第三届晋级赛（公开 writeup，来源见文末）。**不含具体答案**。

---

## 基本信息

- 主办：奇安信集团 盘古石团队
- 频次：2023 起每年一届
- 形式：预选赛（线上）→ 晋级赛（线上）→ 决赛（线下）
- 团队赛：约 100 题，检材量大（多台手机 + 多台电脑 + 服务器 + IoT）
- 工具倾向：商业工具（盘古石取证大师、雷电模拟器）+ 开源（blutter、docker 分析）

---

## 检材构成（2025 晋级赛实测）

| 检材类 | 具体形态 | 备注 |
|---|---|---|
| 多部安卓手机 | 火眼/盘古石导出压缩包 | 常含多角色 |
| 苹果手机 | iTunes backup + 设备镜像 | IDFA、iCloud 关联 |
| Windows 电脑 | E01 镜像 + 可能的内存 | 主角的工作机 |
| Mac 电脑 | E01 或 dmg 镜像 | 含 macOS 应用分析 |
| Linux 服务器 | 多台（启动盘 + 数据盘） | **Docker 集群**是核心考点 |
| IoT 设备 | 智能冰箱、摄像头、路由器 | 厂商专有协议 |
| 独立样本 | APK / EXE / JS / 加密文件 | 含重度逆向 |

## 考察方向分布（2025 晋级赛）

### 1. 手机基础信息（~10%）
- IMSI / IDFA / IMEI / 型号
- 邮箱账号、已卸载应用的卸载时间
- 快递地址、通讯记录
- **特色**：Telegram/QQ 加密数据库解密、群组内部代号

### 2. APK 深度逆向（~20%，盘古石特色）
- 远程控制工具（RDP-like）分析：中继服务器 IP/端口/Key
- 监听/录音 APK：启动延迟、录音阈值、停止条件、加密密钥
- **Flutter 逆向**（用 `blutter` 工具反编译 arm64-v8a 的 `libapp.so`）
- 录音/录像文件加密方式（RC4 / AES / RSA，可能多层嵌套）

### 3. 计算机取证（~20%）
- Build 版本、SID、MAC 地址、默认浏览器
- xshell 远程连接 IP（会话配置文件解密）
- 加密容器密码（VeraCrypt/TrueCrypt）
- 伪造身份证 / 密钥文件
- 反取证软件识别
- Windows 木马分析（C2 IP、复制路径、窃取浏览器数据种类）

### 4. Mac / macOS 取证（~5-10%）
- macOS 加密程序逆向
- PBKDF2 种子、文件头元素顺序、减速机制（故意让解密慢）
- keychain / plist 分析

### 5. Linux 服务器集群（~20%，高分）
- 启动盘 GUID、网关、数据盘文件系统（多盘 LVM/RAID）
- 数据盘密钥文件 / 服务器密码
- **Docker 容器**：镜像数量、端口映射、容器间网络
- Web 应用（投资理财网站）：
  - 框架识别（Django / Flask / Node）
  - 数据库密码、后台地址
  - 用户表、订单表、充值记录 SQL 查询
  - 前端 JS 代码分析（`crypto.js` 函数层）
  - 手机号加密方法还原

### 6. 物联网 / IoT（~10%，盘古石独有特色）
- 智能冰箱：品牌、型号、UUID、存储图片分析
- 视频监控：被修改的录像、隐写信息
- **需要厂商 SDK 或通用 IoT 固件分析技巧**

### 7. 数据分析（~15%，高分密集）
- 层级关系：诈骗集团领导 / 下线结构
- 交易统计：下线提现总额、平均佣金、交易次数增长率
- 地址统计：最常见省份、提现金额
- 净资金流计算
- **本质上是 SQL 或 pandas 查询题**

## 盘古石杯 vs 平航杯 差异

| 方面 | 盘古石杯 | 平航杯 |
|---|---|---|
| 考察深度 | 逆向更深（smali / Flutter / 固件级） | CTF 风，覆盖面广但单题浅 |
| Docker 容器 | 核心考点 | 偶尔出现 |
| IoT | 固定考区 | 很少 |
| 数据分析 | 大量 SQL / pandas 题 | 少 |
| 商业工具 | 强推盘古石取证大师 | 工具不限 |
| AI 友好度 | 中（大量固件/Flutter 需要人类熟手） | 高 |

## AI 可能擅长的部分

| 题型 | AI 得分率预期 |
|---|---|
| 手机基础信息（IMSI/IDFA） | 90% |
| 计算机基础（SID/MAC/浏览器） | 90% |
| 数据分析（SQL / pandas） | 85% |
| APK Java 层字符串提取 | 80% |
| Windows 木马 .NET 逆向 | 80% |
| Flutter / blutter 需要设置环境 | 40-60% |
| macOS 加密程序 | 50% |
| IoT 固件分析 | 20-40% |
| Docker 集群网络拓扑推理 | 50% |

## 坑与建议

- **Flutter 反编译**：必须用 `blutter`（GitHub: worawit/blutter），需要 Dart SDK 对齐版本，环境搭建易坑
- **iTunes backup 加密**：有密码的 backup 要先解密才能看数据
- **Docker 容器**：要在挂载的镜像里找 `/var/lib/docker/overlay2/*/diff/` 里的实际容器数据
- **IoT 厂商专有**：冰箱/摄像头题常需要看固件或 SDK，没有通用套路
- **SQL 题**：盘古石题库里有复杂多表 JOIN 和时间窗口计算，AI 写 SQL 非常顺手
- **Mac 的 E01**：APFS 文件系统，TSK 支持有限，可能要用 `apfs-fuse`
