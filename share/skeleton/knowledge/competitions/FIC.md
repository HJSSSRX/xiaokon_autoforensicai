# FIC 全国网络空间取证竞赛 题型分类

由公安部第三研究所、弘连网络等主办的电子取证赛事。
本分析基于 **2024 第四届线上初赛**（公开 writeup）。**不含具体答案**。

---

## 基本信息

- 主办：公安部第三所、弘连网络（火眼产品方）
- 频次：年度一届（2024 为第四届）
- 形式：线上初赛（40-60 题）→ 线下决赛
- 题型：**填空 + 选择**（关键字得分 + 标准答案得分）
- 工具倾向：**火眼证据分析是主力**（出题方是火眼厂商）
- 容错率低：一次提交，答错不能重试

---

## 检材构成（2024 初赛实测）

最大特色：**企业级虚拟化集群 + 软路由 + 真实 CMS**

| 检材类 | 具体形态 | 备注 |
|---|---|---|
| 多台个人 PC | 涉案技术人员 + 被害人的电脑镜像 | 含 BitLocker 加密分区 |
| ESXi 宿主机 | 整个 VMware ESXi 6.7 镜像 | **核心**，内含多台 VM |
| ESXi 下多台虚拟机 | www（网站）+ data（数据库）+ rocketchat（IM）+ 其他 | 虚拟机需要仿真启动 |
| 软路由镜像 | OpenWrt / iStoreOS | 含 Docker + VPN 代理 |
| 手机（可能） | 按案情延续 | 辅助证据 |

## 考察方向（2024 初赛）

### 1. ESXi 虚拟化平台（~15%，FIC 独有特色）
- ESXi 版本识别 / 安装日期 / 静态 IP
- 数据存储 UUID（**`esxcfg-volume -l` 挂载丢失的 datastore**）
- 创建的虚拟机数量
- root 默认空密码登录（常见考点）

### 2. 服务器渗透与配置（~20%）
- ssh 密码爆破（`hydra`）
- 宝塔面板分析（默认端口、建站目录）
- Web 应用识别（**RuoyiCMS / Java Spring Boot 常见**）
- Nginx 配置分析
- Redis / MySQL 配置参数

### 3. Java Web 后台逆向（~10%）
- 用 `jadx` 反编译 Spring Boot jar
- 读登录 Controller 找密码加密算法
- **盐值**提取（硬编码在 `EncryptionUtils`）
- 算法识别（bcrypt / md5 / sha-1 等）

### 4. Docker 容器分析（~10%）
- 识别运行中的容器
- 容器内部 IP（`172.18.0.x`）
- 容器内服务端口映射
- **MongoDB 在容器里**，需通过 ssh 隧道访问
- Navicat SSH tunnel 连接容器数据库

### 5. MongoDB 数据分析（~5%）
- `rocketchat` 数据库的 `users` 集合
- `rocketchat_message` 集合查聊天记录
- 管理员创建时间、用户角色

### 6. 聊天服务器取证（~5%）
- RocketChat 后台配置：文件上传大小、管理员邮箱
- 账号数量（真实用户 vs bot）
- 在聊天记录中找涉诈网站 URL

### 7. 业务数据 SQL 分析（~15%）
- 注册用户总数、地区分布
- 提现成功金额总计（按 status 过滤）
- 登录 IP 统计
- **与盘古石杯的"数据分析题"高度类似**

### 8. 软路由（OpenWrt/iStoreOS）（~10%）
- 主机名、内核版本、静态 IP
- 网卡名（如 `br-lan`）
- Docker 版本
- **VPN 代理软件**（PassWall2 / OpenClash / ShadowSocksR）
- 工具配置路径（如 `/root/Configs/NasTools`）

### 9. Windows 取证（~10%）
- BitLocker 加密分区解密（密码从其他证据推）
- 浏览器历史（**Chrome 保存的账号密码**常是下个服务的入口）
- 微信聊天记录
- 用户活动痕迹

### 10. 团伙关系与时间线（~5%）
- 利润分配比例
- 收网时间段（**注意 UTC 与本地时间差 8 小时**）
- 未抓获嫌疑人识别（从聊天记录交叉推理）

| 题型 | AI 得分率 | 说明 |

| ESXi 虚拟机枚举 | 80% | 挂载后看配置文件 |

## 坑与建议

- **ESXi 的 datastore 丢失**：从镜像挂载出的 ESXi 可能看不到任何虚拟机，需要 `esxcfg-volume -l` 查 UUID 然后 `-m` 挂载。常见陷阱
- **火眼仿真强依赖**：题目出题方就是火眼厂商，某些题（如 ESXi 启动、虚拟机启动状态）**必须**用火眼仿真
- **UTC 与本地时间差**：FIC 题答案经常混用，一定看清题目要求
- **BitLocker 密码链**：常见套路是"从 A 的 Chrome 保存的密码 → 解 B 的 BitLocker → 找到 C 服务器的密码"。跨检材推理
- **容器内服务端口只监听容器内部网络**（`172.18.0.x`），外部访问必须建 ssh 隧道
- **宝塔面板密码**：如果原密码记不住，可以改配置文件重置
- **Docker 挂载宿主机镜像**：镜像里的 `/var/lib/docker/overlay2/` 是容器文件系统，要挖数据进去找
- **一次提交**：答案不确定时宁可不交也别乱交（答错扣分或零分）
