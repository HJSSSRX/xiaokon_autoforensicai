# 美亚杯 题型分类

全称：中国电子数据取证大赛（"美亚杯"），由美亚柏科协办。
本分析基于 **2024 第十届团体赛**（2025 第十一届刚开赛，尚无公开 WP）。**不含具体答案**。

---

## 基本信息

- 主办：美亚柏科
- 频次：年度一届（2024 为第十届）
- 形式：个人赛 + 团体赛
- 语言：**繁体中文 + 英文**（港澳台风格）
- 题目数：团体赛约 130 题（2024 年）
- 时长：较长（半天以上）
- 题型：**大量单选题 + 判断题 + 少量填空题**（与内地 CTF 风格差别最大）

---

## 题型特色（与内地赛差别巨大）

| 特征 | 美亚杯 | 内地赛（平航杯/盘古石） |
|---|---|---|
| 主体题型 | 选择题 / 判断题 | 填空题 |
| 答题方式 | 可蒙可推理 | 必须给出精确字符串 |
| 检材多样性 | 极高（VR/NAS/Mac/iTunes/USB/SD/Jumpstation） | 中等 |
| 犯罪剧情 | 完整犯罪网络，多角色串联 | 单主角 + 配角 |
| 深度 | smali 级逆向（比平航杯深） | Java/C# 源码级 |
| 时间顺序 | 题目按剧情推进 | 按检材类型 |

---

## 典型剧情架构（2024 团体赛）

剧情式命题，题目沿着犯罪网络展开：

```
Alice（手机/Mac）─────────→ John（手机/Desktop/NAS）
    │                           │
    └────┬──────────────────────┘
         ↓
   Ben（手机/Laptop/USB/SD/Jumpstation）
         │
         ↓
   David（Laptop，虚拟货币中心）
         │
         ↓
   虚拟货币交易网络 → 最终洗钱地址
```

每个角色有多种检材，**检材之间存在强关联**，答错一题可能连带后面错。

## 考察方向分布（2024 团体赛 130 题）

### 1. 移动设备取证（~25%）
- Android bin 文件解析（`.bin` 是美亚柏科 MBOX 格式）
- iTunes backup zip 分析（含密码如 "1234"）
- iPhone zip 镜像
- **通讯记录**：WhatsApp / Facebook / Telegram 群组、消息类型编号、数据库表结构
- **媒体文件**：照片 EXIF、GPS、多媒体文件时间线
- **应用痕迹**：Foodpanda 外卖订单、支付方式、常用店铺

### 2. APK 深度逆向（~15%，smali 级别）
- 识别 AhMyth RAT / 类似开源 RAT 工具
- **smali 代码阅读**（不是 Java！必须看 .smali）：
  - 关键类路径（如 `ahmyth/mine/king/ahmyth/`）
  - `b.smali` 图像处理、`g.smali` 录音、`h.smali` 文件传输
  - MediaRecorder、TimerTask、ConnectionManager 的 smali 实现
- 图像压缩比率、字节数组操作顺序
- 权限声明（AndroidManifest）、未请求的权限
- 主活动布局文件、Public ID

### 3. 加密破解（~5-10%）
- **PBKDF2 + AES 加密文件**：
  - 读 `encrypt.py` 源码推 IV / SALT
  - 计算派生密钥
  - 推算 `count` 参数变化对加密的影响
- 加密压缩包（7z/RAR）字典爆破
- SD 卡加密密码

### 4. 计算机取证（~20%）
- Windows 桌面（John_Desktop.e01）
- Windows 笔记本（Ben_Laptop.zip）
- **Mac E01**（Alice_Macbook.e01，macOS Monterey）：
  - 计算机名、macOS 版本、EN0 MAC 地址
  - 浏览器痕迹、下载历史
  - TeamViewer ClientID / 连接记录
- **Jumpstation**（跳转机）：远程桌面软件、连接日志

### 5. 虚拟货币追踪（~10%，美亚杯特色）
- IDFC 交易追踪（类似 USDT 的虚构币）
- 多层洗钱钱包地址识别
- AI 深度伪造视频诈骗（交易哈希定位）
- 资金最终归集地址

### 6. 暗网 / VPN / Tor（~5%）
- Kali Linux 中启动 Tor 命令
- Firefox 与 Tor 网络关系
- OpenVPN 使用天数、加密算法
- `.onion` 书签数量
- Port 9151 特征

### 7. 网络取证（~10%）
- pcap 分析：nmap 扫描命令、reverse shell 获取
- HTTP/TLS 协议分析（TLS 降级、DNS 外传）
- Web 应用防火墙日志分析
- 十六进制包头解析（源 IP 提取）
- 邮件头伪造识别

### 8. 特殊设备取证（~5%，美亚杯独有）
- **VR 设备**（Meta Quest 3 zip）：
  - 用户数、默认 IM 软件、游戏版本
  - DCIM 文件夹分析
  - Wi-Fi / 蓝牙能力
- **USB / SD 卡**：JPG 对比、隐写分析、加密密码
- **NAS 存储（RAID）**：
  - RAID 类型识别
  - 文件系统（EXT3/EXT4/Btrfs）
  - 网络驱动器盘符
  - 违禁内容识别（"Child PORN PHOTO" 标识）

### 9. 黑客技术基础（~5%）
- Google 高级搜索运算符
- Python `http.server` 启动 HTTP 服务
- 反弹 shell（nc / bash / python pty.spawn）
- SUID 权限（`find -perm -4000`）

## AI 得分率预期

| 题型 | AI 得分率 | 说明 |
|---|---|---|
| 选择题 | **70-80%** | 4 选 1 纯蒙也有 25%，加上推理能到高 |
| 判断题 | **75-85%** | 2 选 1，配合证据链推理 |
| 填空题 | 50-70% | 需要精确字符串 |
| smali 阅读 | 70% | LLM 读 smali 比 Java 稍弱，但套路题能答 |
| PBKDF2 / AES 复现 | 85% | 给源码 + 写 Python 脚本是 LLM 强项 |
| 虚拟货币地址追踪 | 40% | 需要链上 API 查询（离线困难） |
| VR / NAS 硬件题 | 50% | 工具支持有限 |
| 网络取证（pcap） | 80% | tshark + LLM 读协议 |
| 黑客常识题 | 95% | 纯知识点 |
| 时间线 / GPS 推理 | 60% | 需要整合多个证据 |

**关键优势**：选择题和判断题占比 ~70%，AI 的"**能识别候选答案中的错误项**"能力让得分率远高于填空题比赛。

## 坑与建议

- **繁简体字符**：答案可能用繁体，填空题要注意
- **`.bin` 格式**：美亚柏科 MBOX 格式，需要他们的解析工具或火眼
- **iTunes backup 加密**：2024 年给了密码 "1234"，2025 可能变
- **smali 阅读**：不能直接用 jadx 反编译，要看 `apktool d` 出的 smali，LLM 读的效果比 Java 弱 20%
- **剧情关联**：前面题错答会导致后面题找不到正确检材
- **macOS APFS**：TSK 对 APFS 支持不完美，可能要用 `apfs-fuse`
- **VR 设备文件结构**：Meta Quest 3 的 Android 内部存储结构和普通手机略有差异
- **NAS RAID 重组**：多 E01 镜像要用 `mdadm --assemble` 或专用工具重组阵列

---

## 与盘古石杯的互补关系

如果按"一次学透"的顺序建议：
1. **先打平航杯** —— 建立基础工具链、熟悉国内赛风格
2. **再打盘古石杯** —— 练深度逆向（Flutter / Docker / smali）
3. **最后打美亚杯** —— 挑战大剧情、多设备关联、国际题风

三个比赛的 playbook 重合度约 60%，剩下 40% 是各自特色。
