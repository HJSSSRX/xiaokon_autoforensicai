## 📋 强制字段清单

每一道题的答案**必须**包含以下 **5 个字段**，缺一不可：

| 序号 | 字段名 | 格式要求 | 说明 |
|------|--------|----------|------|
| 1 | **题号 + 题目简称** | `## Q<NN> — <简称>` | 二级标题，题号用实际数字，简称自拟（≤10字） |
| 2 | **题目原文** | `> <从 questions.md 复制的完整题目>` | 引用块，一字不差复制 |
| 3 | **答案** | `**答案**：\`<值>\`` + **置信度** 高/中/低 | 反引号包裹值，另起一行标注置信度 |
| 4 | **解析** | 分四步：识别/提取/分析/验证，含可复现命令 | 详细步骤，命令用 ```bash 包裹 |
| 5 | **不作弊声明** | 四要素：数据来源/未访问外部/工具列表/脚本留存 | 固定格式，见下方模板 |

---

## 📝 单题模板（复制粘贴使用）

```markdown
## Q<NN> — <题目简称>

> <从 questions.md 一字不差复制题目>

**答案**：`<具体答案值>`
**置信度**：高/中/低

### 解析

#### 识别
<说明本题要找什么，关键信息点>

#### 提取
<说明从哪个文件/路径提取，用什么工具>

#### 分析
<说明如何处理数据，推理过程>

#### 验证
<说明如何验证答案正确性，提供可复现命令>

```bash
# 可复现命令
<命令>
```

### 不作弊声明
- **数据来源**：本题所有数据均来自检材盘 `/mnt/a/` 及挂载点
- **未访问外部**：未使用任何外部网络资源或预知答案
- **工具列表**：volatility3, sqlite3, strings, grep, bash, python3
- **脚本留存**：相关脚本留存于 `cases/2026平航杯团体赛/artifacts/`
```

## 🎯 置信度判定标准

| 等级 | 判定标准 | 示例 |
|------|----------|------|
| **高** | 证据链完整，工具直接输出，无歧义 | 文件哈希值、数据库字段值 |
| **中** | 证据链基本完整，需人工推断，可能有其他解释 | 进程关联性、行为意图 |
| **低** | 证据链不完整，多解，关键信息缺失 | 部分解码内容、损坏文件 |

## 📁 产出位置

- **批次文件**：`cases/<比赛>/wp_batches/Q<NN>-Q<MM>.md`（每5题一份）
- **命名规范**：`Q<起始题号>-Q<结束题号>.md`
- **文件抬头**：必须包含比赛名称、批次、解题人、时间

---

## ⚠️ 死规矩（违反视为回答无效）

1. **缺字段**：若输出缺5字段中任一，用户有权要求立刻返工
2. **格式错**：不使用模板格式，用户可拒收并记录到 `worklog/AI_DIAGNOSTICS.md`
3. **无命令**：解析部分无可复现命令，答案无效
4. **不声明**：缺不作弊声明，视为作弊
5. **乱位置**：答案不放批次文件，放其他位置无效

## 🔧 修改指引

### 用户想修改"答案必含什么字段 / 格式"时去哪改？

- **加/删字段** → 改 `.windsurf/rules/project-rules.md` §"答案输出格式"
- **改模板细节**（排版、证据块样式） → 改本文件 `WP_FORMAT.md`
- **加新"死规矩"** → 改 `WP_FORMAT.md` §"死规矩"表

### AI 如何使用本契约

1. **答题前**：先读一遍本契约，记住5字段
2. **答题中**：严格按照模板格式输出
3. **答题后**：自查是否缺字段、格式对错
4. **被拒时**：立刻返工，不争辩，记录到 diagnostics

## 📊 常见错误自查表

| 错误类型 | 表现 | 修复方法 |
|----------|------|----------|
| 缺题目原文 | 无引用块或内容不全 | 复制完整题目到引用块 |
| 答案格式错 | 不用反引号或缺置信度 | 改为 `\`值\`` + 另起一行置信度 |
| 解析太简 | 少于4步或无命令 | 补全四步，添加可复现命令 |
| 声明不全 | 少四要素中任一 | 补充完整四要素 |
| 位置错误 | 放错目录或文件名 | 移动到正确批次文件 |
