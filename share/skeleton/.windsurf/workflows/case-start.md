---
description: 新比赛开始时的标准流程（替代让 AI 盲跑）
---

当用户说 /case-start <比赛名> <检材路径> 时，按以下步骤执行，每步做完前不准跳下一步。

1. 确认 `cases/<比赛名>/` 目录不存在；若存在，告诉用户并问是否继续（避免覆盖旧数据）。
2. 创建目录结构：
   ```
   cases/<比赛名>/
   ├─ artifacts/
   ├─ wp_batches/
   └─ evidence -> <检材路径>  (Windows 符号链接或记录路径到 NOTES.md)
   ```
3. 读取题目：
   - 如果检材里有 `*.xlsx`，用 Python+openpyxl 转换成 `questions.md`（每题一段）
   - 如果没有，询问用户题目来源
4. 读 `@/SESSION_START.md` 和 `@/PLAYBOOK.md` 复习套路。
5. 读 `questions.md` **全文**，把题目分类（Windows / Android / 流量 / 内存 / 恶意样本 / 跨类）。
6. 把分类结果写入 `cases/<比赛名>/TASKS.md`：题段 → 推荐机器 → 预估用时。
7. 报告给用户：
   - 总题数
   - 分类统计
   - 建议的三机分工（或单机顺序）
   - 第一批 5 题打算怎么做
8. **等用户确认**后再开工。不要擅自开始跑工具。

注：第 1-3 步的目录创建和文件转换可自动执行，但第 7 步的方案报告必须等用户批准。
