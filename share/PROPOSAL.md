# ForensicAI — 项目策划书

**智能电子数据取证平台：从 AI 自动化到取证教育的全栈解决方案**

---

## 一、项目概述

### 项目名称

**ForensicAI** — 面向电子数据取证的智能辅助与教育平台

### 一句话介绍

一个**离线可用、模型无关、开源协作**的电子数据取证 AI 辅助系统，同时也是一个**带着新手从零到精通**的取证教育平台。

### 项目定位

```
                    离线/断网
                       ▲
                       │
         ForensicAI ★  │  （无竞品）
                       │
  教育 ◄───────────────┼───────────────► 自动化
                       │
        传统培训        │     通用CTF Agent
        (盘古石CPFP)   │     (ctf-agent/Squid Agent)
                       │
                       ▼
                    联网/云端
```

**核心差异化**: 同时满足"离线自动化"和"取证教育"两个需求。现有方案要么只做自动化（且依赖云端），要么只做培训（且线下为主）。ForensicAI 是唯一在两个维度同时覆盖的解决方案。

## 二、行业背景与痛点

### 2.1 电子取证行业现状

- **人才缺口**: 国内电子数据取证从业者约 3-5 万人，但公安、检察、法院系统对取证人才的需求远超现有供给。CISP-F 持证者不足 5000 人。
- **培训成本高**: 盘古石 CPFP 认证培训 ¥3000-8000/期，看雪数字取证训练营 ¥2000+/180天。培训供给有限，更新周期长。
- **比赛是核心评价场景**: FIC（Forensic Innovation Cup）、平航杯、盘古石杯、美亚杯、网鼎杯等比赛是衡量取证能力的主要渠道，直接影响职称评定和招投标。
- **比赛迎来 AI 变革**: 2025-2026 年，AI 辅助已经开始改变 CTF 和取证比赛的格局。但中文取证领域的 AI 方案尚属空白。

### 2.2 行业痛点

| 痛点 | 影响对象 | 现有解决方案 | 局限 |
|------|---------|------------|------|
| **取证比赛提分难** | 参赛团队 | 纯人工经验积累 | 耗时长、不可复制 |
| **培训体系封闭且昂贵** | 新手/学生 | 付费培训班 | 价格高、更新慢、覆盖面窄 |
| **线下赛断网限制** | 比赛组织者/参赛者 | 无 | 现有 AI 方案全部依赖联网 |
| **知识碎片化** | 全行业 | 博客/公众号/论坛 | 质量参差、难以系统化 |
| **取证工具链复杂** | 工程师 | 手动安装配置 | 环境冲突频发、新人上手慢 |

## 三、解决方案

### 3.1 产品形态

ForensicAI 是一个**开源的文件型知识系统**（不是独立的 SaaS 应用），它通过结构化的 Markdown + YAML 文件为任何 AI 编程助手提供取证领域的专业知识和工作流。

```
ForensicAI 不是一个"新的 AI 工具"
ForensicAI 是让"任何 AI 工具"都能做取证的"大脑"
```

### 3.2 核心功能模块

#### 模块 1: 智能取证知识库

- **结构化题库**: 每道已解题目包含元数据、答案、人工可复现的解题步骤、工具配置、教育引导
- **双模式索引**: RAG 向量检索（中高端模型）+ 确定性 SOP 脚本（低端模型/纯人工）
- **分类标签体系**: 两级分类覆盖 7 大证据类型、40+ 知识子类
- **持续积累**: 每做一道题，知识库就增长一分

#### 模块 2: 多模型自动取证

- **模型无关**: 支持 Claude、GPT、DeepSeek、Qwen 等任何模型，包括本地部署
- **离线运行**: Ollama/vLLM + 本地 embedding + 预构建 FAISS 索引 = 完全断网可用
- **策略配置**: 根据网络环境、模型能力、比赛类型自动选择最优策略
- **答案验证**: 自动格式校验 + AI 自检 + 人工抽检三层验证

#### 模块 3: AI Agent 适配层

- **通用入口 (AGENTS.md)**: 一份核心规则，所有 Agent 共享
- **专用适配器**: Windsurf / Cursor / Claude Code / OpenClaw 等各自的配置文件
- **行为契约**: 严格定义输出格式、禁止行为、协作规则

#### 模块 4: 教育引擎

- **三级教学**: 手把手引导 → 提示引导 → 独立练习
- **苏格拉底式交互**: AI 不给答案，引导学生自己发现
- **自动学习路径**: 基于题目难度和知识依赖关系自动生成
- **低模型友好**: 教学脚本预编写，7B 参数模型即可驱动

#### 模块 5: 工具链自动化

- **一键部署**: 从裸机到取证就绪，自动安装 30+ 工具
- **跨平台**: Windows / Linux / WSL 全覆盖
- **离线安装包**: 预下载所有依赖，断网也能部署
- **哈希校验**: 防篡改、防供应链攻击

#### 模块 6: 协作与共享

- **开源知识库**: 公共仓库积累社区贡献的题解
- **私有扩展**: 敏感内容可保留在团队内部
- **贡献流程**: PR + 审核 + 自动重建索引
- **版本管理**: 语义版本号 + CHANGELOG

## 四、技术优势与创新点

### 4.1 与现有方案的差异化对比

| 维度 | ForensicAI | ctf-agent (Veria) | Squid Agent (SPL) | 盘古石培训 |
|------|-----------|-------------------|-------------------|----------|
| **离线运行** | ✅ 核心设计 | ❌ 依赖云端 API | ❌ 依赖云端 API | ✅ 线下授课 |
| **取证特化** | ✅ 深度特化 | ❌ 通用 CTF | ⚠️ 取证最弱 | ✅ 深度特化 |
| **模型灵活** | ✅ 任意模型 | ⚠️ Claude+GPT | ⚠️ Claude | N/A |
| **教育功能** | ✅ 原生支持 | ❌ 无 | ❌ 无 | ✅ 核心功能 |
| **开源免费** | ✅ | ✅ | ⚠️ 部分 | ❌ ¥3000+/期 |
| **知识积累** | ✅ 结构化 | ❌ 无持久化 | ⚠️ RAG 但不结构化 | ⚠️ 课件形式 |
| **多人协作** | ✅ Git-based | ✅ Swarm | ❌ 单机 | ⚠️ 课堂 |
| **中文支持** | ✅ 原生中文 | ❌ 英文 | ❌ 英文 | ✅ 中文 |

### 4.2 关键技术创新

1. **双模式索引 (Dual-Mode Index)**
   - 业界首创：同一知识库同时支持 RAG 向量检索（强模型）和确定性 SOP 脚本（弱模型/人工）
   - 解决了"本地小模型无法做复杂取证推理"的核心难题

2. **训练-执行分离架构 (Train-Execute Split)**
   - 用强模型（云端 Claude）在平时训练中产出高质量知识
   - 弱模型（本地 7B-32B）在比赛中消费这些知识执行任务
   - 类似 Cyber-Zero (Amazon) 的"用 writeup 训练 agent"思路，但更实用

3. **教育即副产品 (Education as Byproduct)**
   - 每道训练题自动产出教育内容（引导脚本、常见错误、学习路径）
   - 教育模式不是单独开发的，而是知识库的天然投影

4. **苏格拉底式 AI 教学**
   - 学术研究 [CSAW 2025, arxiv 2603.21551] 证实：60% 的学生把 AI 当答案机器，什么都没学到
   - ForensicAI 的教育模式强制引导式交互，确保学习效果

## 五、实战验证

### 5.1 FIC2026 电子取证比赛实测

| 指标 | 数据 |
|------|------|
| 比赛 | FIC2026 全国电子数据取证比赛 |
| 最终得分 | **367 / 600 (61.2%)** |
| AI 运行时间 | 1 小时（因 token 额度限制中断） |
| Token 成本 | ~$10 (约 ¥72) |
| AI 独立解题 | 33 / 52 题 |
| 每题成本 | ¥2.2/题 |
| 每分钟产出 | 0.55 道题 |
| 本届最高分 | ~556 / 600 |

### 5.2 关键发现

- **AI 在 1 小时内完成了人类 4 小时工作量的 60%**
- **成本极低**: 全部花费不到一杯咖啡
- **协作效果显著**: AI 解 33 题 + 人工修正 5 题 + 人工补充 7 题 = 最终 45 题
- **瓶颈已识别**: token 额度（非能力）是主要限制，如果跑满 4 小时预估可达 480-520 分

### 5.3 改进后预期

| 场景 | 预期得分 | 成本 | 条件 |
|------|---------|------|------|
| 当前方案 | 367/600 | ¥72 | 1h AI + 人工 |
| 改进后 (云端) | 480-520/600 | ¥250-350 | 4h × 3 并行 + 验证层 |
| 改进后 (本地) | 400-450/600 | ¥0 (电费除外) | 4h × 本地 32B + RAG |

## 十二、关于"AI 预训练是否已经知道答案"的澄清

一个常见的质疑是："如果 AI 能解出这道题，说明它在预训练时已经见过答案，那训练知识库有什么意义？"

**事实澄清:**

1. **中文取证比赛题目极少出现在公开互联网上**。FIC/平航杯/盘古石杯的题目和答案不在 AI 的训练数据中。
2. **AI "知道的"是方法论，不是具体答案**。它知道"怎么用 RECmd 分析注册表"，但不知道某个特定检材的注册表里有什么。
3. **学术研究 [arxiv 2604.13997] 确认**: LLM 的基准测试表现是"记忆"和"泛化"的混合，在新领域（如中文取证）以泛化为主。
4. **知识库的价值是方法可复现**：即使 AI 被换成另一个模型，或者在断网环境用弱模型，知识库中的 SOP 脚本仍然可以指导解题。

**结论**: 训练产出的知识库不是在"记录 AI 已经知道的答案"，而是在**将 AI 的推理过程固化为任何人/任何模型都可复现的标准操作程序**。
