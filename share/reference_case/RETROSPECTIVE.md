## 3. 做错了什么 / 卡了多久

### ❌ C03/C08 — 邮件内容挖掘失败 (浪费 ~40 分钟)
- **问题**: 黄金商家联系方式和勒索软件联系方式都在邮件中，但没找到
- **根因**: deepin-mail 的 LevelDB + Foxmail 缓存结构不够熟悉；邮件搜索策略太粗放（只 grep 关键词）
- **教训**: 应该先建邮件全量索引（发件人/主题/正文/附件），再做精确搜索

### ❌ C04 — RSA 加密推广图 (浪费 ~30 分钟)
- **问题**: 找到了 `推广设计图.png.enc`、`public.txt`（含 n 和 e）、`加密图片查看.html`，但没找到私钥
- **根因**: 私钥可能藏在其他位置（如浏览器缓存、密钥管理器），搜索范围不够广
- **教训**: RSA 解密题，关键是私钥；应该在发现 public.txt 后立刻全盘搜索私钥特征串（`-----BEGIN RSA PRIVATE KEY-----` 或 `d=` 等）

### ❌ S08/S13-S15 — TiDB 数据库访问受阻 (浪费 ~20 分钟)
- **问题**: TiDB 运行在 LXC 容器中，需要 sudo 进入容器，但 WSL 环境无 sudo 密码
- **根因**: 离线分析时没有直接读 TiDB 数据文件的工具链
- **教训**: 应该提前准备 TiDB 离线数据文件解析方案，或尝试直接读 `/var/lib/lxc/mytidb/rootfs/` 下的数据文件

### ❌ M10 — DJI 飞行日志加密 (浪费 ~25 分钟)
- **问题**: DJI Flight Record V12+ 全段 AES-CTR 加密，无公开解密工具
- **根因**: DJI 2026 版加密升级，开源工具链只支持到 V11
- **教训**: 遇到厂商私有加密且无 key 时，应该更快放弃，转向间接证据（如 DJI 账号云端数据、GPS 照片 EXIF）

### ❌ I01/I02 — 互联网题几乎空白
- **问题**: 售卖卡密的群组ID 和备份数据库视频图片完全未做
- **根因**: 时间分配不合理，互联网题放到最后没时间了
- **教训**: 互联网题通常分值高且可能需要跨检材关联，不应放到最后

### ⚠️ PowerShell → WSL 引号地狱 (累计浪费 >30 分钟)
- **问题**: 复杂 bash 内联到 PowerShell 反复出错
- **已修复**: 比赛中后期改为写脚本文件再执行，大幅减少出错
